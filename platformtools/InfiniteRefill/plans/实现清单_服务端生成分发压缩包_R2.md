# 实现清单：服务端生成分发压缩包（R2）

本清单用于在实现阶段逐项落地你要求的链路：

- 管理员一次申请 N 个平台密钥（`USER_KEY` 或 `UPLOAD_KEY`）
- 服务端为每个密钥生成一个**标准 zip 密码**的加密压缩包（数字命名）
  - 约束：用户拿到 `1.zip` 必须能用 7zip/系统解压，输入密码（密码=对应平台密钥）即可打开
- 生成 `分发清单.txt`（每行：`压缩包名：对应密钥`）
- 产物写入 R2，接口返回下载链接（支持你离线转交）

对应设计入口：[`server/DESIGN.md`](../server/DESIGN.md:183)。

---

## A. Cloudflare 资源准备

- R2 Bucket
  - 用途：存放 `batches/<batch_id>/*.zip` 与 `batches/<batch_id>/分发清单.txt`
  - 生命周期：建议配置自动过期（例如 1~7 天）
- Zero Trust Access 或 WAF IP 白名单
  - 保护 `/admin/*`

---

## B. 服务端接口（Workers）

### B1) `POST /admin/packages/issue`

关键实现选择（已按你确认修改为 presigned URL 方案）：

- 生成 **R2 presigned GET URL**（短期有效，例如 60 分钟）并返回给管理员下载。
- Worker 使用 `aws4fetch` 做 SigV4 query sign（Cloudflare 文档示例可行）。

对应 Cloudflare 文档参考：

- R2 presigned URLs：[`r2.api.presigned-urls`](https://developers.cloudflare.com/r2/api/s3/presigned-urls/:1)
- aws4fetch 示例：[`r2.examples.aws4fetch`](https://developers.cloudflare.com/r2/examples/aws/aws4fetch/:1)

注意：

- 仍然要求 `/admin/*` 走 Zero Trust Access 或 IP 白名单。
- 生成 URL 本身不暴露 R2 API secret，但 URL 属于临时授权凭据，应设置短 TTL。

输入：`type,count,label,bind_pool_size,server_url,ttl_minutes` 等。

输出：

- `batch_id`
- `packages[]`：`name`、`key`（明文，仅一次）以及 `download_url`
- `manifest`：`分发清单.txt` 的 `download_url`

实现步骤：

1) 校验管理员权限（[`server.index.requireAdmin()`](../server/src/index.ts:266)）。
2) 生成 N 个密钥（24 bytes base64url，形如 `k_...`），服务端仅存 hash：
   - user: 插入 `user_keys`
   - upload: 插入 `upload_keys`
3) 为每个 key 构造 zip 内容：
   - `无限续杯配置.env`（含 `SERVER_URL` 与 key）
   - `README.txt`
4) 生成 zip，并按 key 设置密码（严格 zip 密码兼容性需选定库）。
5) 上传至 R2：
   - `batches/<batch_id>/<i>.zip`
6) 生成 `分发清单.txt` 并上传：
   - `batches/<batch_id>/分发清单.txt`
7) 生成下载链接：
   - 若使用 R2 public bucket：可直接拼 URL（不推荐）
   - 若私有：用 signed URL（推荐，短期有效）

## C. 打包与加密（难点清单）

你要求：压缩包可用常见工具直接输入密码打开（密码=key）。

需要在实现前确认（实现阶段必须验证）：

- 选用的 zip 库是否支持：
  - **标准 zip 密码**（能被 7zip/系统解压识别）
  - AES-256 或 ZipCrypto（二者都可能；关键是兼容性）
  - 在 Workers runtime 内生成 zip（无 Node 原生依赖）

候选库（待在实现阶段做 PoC 验证）：

- `zip.js`：文档显示支持 `ZipWriter` 生成 zip；需进一步确认其“password/encryptionStrength”在 Workers 运行时的兼容性。

> 你已明确不接受外层 AES-GCM `.zip.enc` 兜底，因此实现阶段必须先做 PoC：Workers 内生成一个带密码的 zip，确保 Windows 资源管理器/7zip 可打开。

---

## D. 安全与防撞库

按你已确认策略：

- `/v1/*`：WAF/Rate Limiting 基础限速 + 401/403 失败增强限速
- `/admin/*`：Zero Trust Access 或 IP 白名单 + 更低频率限速
- Worker 兜底：key 格式校验 + 统一错误响应体（仍保留 missing=401、invalid=403）

---

## E. 兼容性与数据边界

- 分发包只包含平台密钥与本项目配置，不包含任何第三方 token。
- `分发清单.txt` 属于高敏文件：
  - 只允许管理员下载
  - 短期有效
  - R2 生命周期自动删除
