# 本地客户端（curl + 中文命名）

本目录提供三类本地客户端脚本，全部使用 `curl` 调用服务器接口，并采用**中文文件名/中文提示**，面向小白。

对应设计：[`server/DESIGN.md`](../server/DESIGN.md:1)

---

## 0. 通用约定

- 服务器地址：例如 `https://xxx.workers.dev`
- 密钥放在 Header：
  - 普通用户：`X-User-Key: <USER_KEY>`
  - 热心群众：`X-Upload-Key: <UPLOAD_KEY>`（也可用于普通用户接口，等价于 `USER_KEY`）
  - 维修者：`X-Upload-Key: <UPLOAD_KEY>`（与热心群众同凭据体系，但主要调用 `/v1/repairs/*`）
  - 超级管理员：`Authorization: Bearer <ADMIN_TOKEN>`

### 0.1 JSON 解析依赖（macOS 小白不装 jq 也能跑）

- 本仓库的 macOS/Linux `.sh` 脚本现在**默认不再硬依赖** `jq`。
- 统一的 JSON 小工具见：[`客户端/_lib/json.sh`](_lib/json.sh:1)
- 优先级：`python3`（推荐） > `osascript/JXA`（macOS 自带） > `jq`（兜底）

### 1.1 自动清理（仅删 401）

目录：[`客户端/普通用户/自动清理`](普通用户/自动清理:1)

- Windows（手动一键，不需要 Python）：[`客户端/普通用户/自动清理/一键清理_仅删401.bat`](普通用户/自动清理/一键清理_仅删401.bat:1)
- Windows（计划任务/无暂停）：直接给 [`客户端/普通用户/自动清理/一键清理_仅删401.bat`](普通用户/自动清理/一键清理_仅删401.bat:1) 传入 `nopause`（以及可选 `apply`）即可

清理后自动补齐（可选）：
- `apply` 且确实删除文件后，会按配置文件：[`客户端/普通用户/状态/无限续杯配置.env`](普通用户/状态/无限续杯配置.env:1)
  - `TARGET_POOL_SIZE` 目标数量
  - `AUTO_REFILL_AFTER_CLEAN=1` 开启后：若不足配额将自动发起补齐请求
- 补齐请求结果追加写入：[`客户端/普通用户/状态/清理后补齐结果.jsonl`](普通用户/状态/清理后补齐结果.jsonl:1)（来自 `POST /v1/refill/topup` 的原始响应，便于你排查服务端返回）

续杯触发规则（单次续杯脚本内实现）：
- 当一组账号里 `401+429 >= 8`，或当前本地账号数量不足目标配额时，触发 `POST /v1/refill/topup`
- 触发后会删除本地 `status_code ∈ {401,429}` 对应的 json 文件（删除前会在 out/ 里备份）
- 上报给服务端的每个账号状态包含 `status_code`：
  - `401` 表示失效
  - `429` 表示配额用光（服务端可据此延迟一周再放出给其他人用）
- macOS/Linux（手动一键，不需要 Python）：[`客户端/普通用户/自动清理/一键清理_仅删401.sh`](普通用户/自动清理/一键清理_仅删401.sh:1)

说明：
- 默认 DryRun，不会删除；加 `apply` 才会删除。
- 只删除 HTTP 401 的文件，其余一律不删。

## 2. 热心群众客户端（上传/上报）

目录：[`客户端/热心群众`](热心群众:1)

热心群众只有一个功能：**提交 JSON**（支持“单文件/文件夹”，并带合规检查）。

- Windows：[`客户端/热心群众/提交JSON.bat`](热心群众/提交JSON.bat:1)
- macOS/Linux：[`客户端/热心群众/提交JSON.sh`](热心群众/提交JSON.sh:1)

用法：

```bash
# 单文件
提交JSON.bat https://127.0.0.1:8787 你的上传密钥 register .\register.json

# 文件夹（提交该目录下所有格式正规的 .json）
提交JSON.bat https://127.0.0.1:8787 你的上传密钥 report .\reports\
```

payload 示例（register）：

```json
{
  "accounts": [
    {
      "email_hash":"<sha256_hex>",
      "account_id":"<可选>",
      "seen_at":"2026-02-28T00:00:00Z",
      "auth_json": {"type":"codex","access_token":"..."}
    }
  ]
}
```

说明：
- `auth_json` 为可选敏感字段，仅当你需要让服务端通过 `POST /v1/refill/topup` 下发账号文件时才上传。
- 服务端会用 `ACCOUNTS_MASTER_KEY_B64` 加密存储；`/admin/backup/export` 仍不会导出任何 token 字段。

payload 示例（probe-report）：

```json
{
  "reports": [
    {"email_hash":"<sha256_hex>","account_id":"<可选>","status_code":401,"probed_at":"2026-02-28T00:00:00Z"}
  ]
}
```
