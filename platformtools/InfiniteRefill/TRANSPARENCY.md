# 源码透明说明（给用户看的）

本仓库包含两大块：

1) 本地客户端工具（[`客户端/README.md`](客户端/README.md:1)）：
   - 三类客户端脚本（普通用户 / 热心群众 / 超级管理员）
   - 其中“自动清理”“单次续杯”在满足条件时会**备份并删除/替换**本地 `auth-dir` 下的认证 JSON（且会输出报告/备份目录）
2) 无限续杯服务器（[`server/README.md`](server/README.md:1)）：Cloudflare Workers + D1。
   - 负责 key 鉴权/分发/接收上报
   - 客户端侧已做合规检查：热心群众提交 JSON 时会跳过包含敏感 token 字段的文件

本文件目的：让使用者能快速定位“哪些脚本会改本地文件、哪些不会、需要什么参数才会生效”。

## B. 可能会删除/替换本地文件的部分

### 1) 401 清理（只在 apply 时删除）

普通用户分发版（不需要 Python）：

- Windows：[`客户端/普通用户/自动清理/一键清理_仅删401.bat`](客户端/普通用户/自动清理/一键清理_仅删401.bat:1)
- macOS/Linux：[`客户端/普通用户/自动清理/一键清理_仅删401.sh`](客户端/普通用户/自动清理/一键清理_仅删401.sh:1)

特征：

- 默认 DryRun，不会删除；加 `apply` 才会删除。
- 只删除探测到 `wham/usage` 返回 HTTP 401 的文件。
- 删除前会备份到输出目录（脚本会打印备份位置）。

### 2) 单次续杯（探测 -> topup -> 落盘 -> 备份并删除 401/429）

- Windows：[`客户端/普通用户/单次续杯.bat`](客户端/普通用户/单次续杯.bat:1)
- macOS/Linux：[`客户端/普通用户/单次续杯.sh`](客户端/普通用户/单次续杯.sh:1)

特征：

- 会探测一组本地账号的可用性，并在触发阈值时请求服务端补齐。
- 触发补齐后，会对 `status_code ∈ {401,429}` 的本地 JSON 进行备份并删除（随后将服务端返回的新账号写入 `auth-dir`）。

### 3) 旧版（需要 Python，仍在仓库中）

这些脚本仍可能读取/删除本地文件，且依赖 Python；已被“curl + 中文客户端”替代，仅用于回溯历史：

- 一键入口：[`UserClient/oneclick-direct-probe-clean-401.bat`](UserClient/oneclick-direct-probe-clean-401.bat:1)
- 计划任务 wrapper：[`UserClient/auto_clean_once.bat`](UserClient/auto_clean_once.bat:1)
- 实际执行逻辑：[`UserClient/codex_direct_probe_cleanup.py`](UserClient/codex_direct_probe_cleanup.py:1)

## 使用者建议（降低误操作风险）

- 任何涉及删除/替换本地文件的脚本，都应该先跑一次 DryRun。
- 尽量在复制的测试目录中演练。
- Windows 的 `auth-dir` 位于 `C:\Users\...` 时，写入/删除可能需要管理员权限。
